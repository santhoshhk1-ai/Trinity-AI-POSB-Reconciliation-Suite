About Trinity AI Suite

“Observe · Self-Learn · Test · Apply” — Ekalavya Soft

Trinity AI Suite is a purpose-built reconciliation toolset developed for India Post POSB (Post Office Savings Bank) operations, designed to automate and simplify the daily financial reconciliation workflow at Head Post Offices.

The suite bridges the gap between three critical data sources — the HO Cash Book, Finacle IT2.0 GL Reports, and Subordinate Office (SO) Summary PDFs — and produces accurate, colour-coded mismatch reports in seconds, replacing hours of manual Excel work.

What It Does

The suite consists of 5 integrated tools, each addressing a specific stage of the reconciliation pipeline:

# Tool Role
1 Finacle GL Compiler v2.2 Parses raw Finacle IT2.0 Transaction GL-wise reports and compiles them into office-wise, account-code-wise Excel data
2 SO Summary Parser v3.47 Converts SO Summary PDFs into structured Excel format using intelligent office-name matching
3 Mismatch Reporter v20.5 Cross-compares Cash Book, Finacle GL, and SO Summary data; produces colour-coded discrepancy reports per office and account code
4 Monthly Reconciliation Consolidator v2.4 Aggregates daily mismatch reports into a monthly workbook with summaries, pivots, and office scorecards
5 HO Office-Mapping Creator v3.1 Generates the office-name mapping reference sheet by intelligently matching SO names across data sources

Key Highlights

  • Zero manual intervention: Load your files, click Run, get results instantly.
  • Intelligent name matching: Handles spelling variations, abbreviations, and multi-token office names.
  • BO figures handled correctly: Branch Office amounts are correctly merged into HO totals.
  • Encrypted branding assets: Professional animated logos secured with AES-128 Fernet.
  • Works completely offline: Zero internet requirements, zero cloud reliance, no subscriptions.
  • Windows 10/11 compatible: Installs directly to C:\Trinity AI POSB Reconciler v3.0.
  • Auto-backup on update: Patch installer backs up your current versions before overwriting.

Setup & Deployment Guide

How to initialize your free hosting workspace in under 5 minutes.

1

Initialize Web Repository

Create a public repository on GitHub. Push this web directory (`index.html` and `style.css`) to the `main` branch.

git init
git add .
git commit -m "Initialize Trinity Hosting Page"
git remote add origin https://github.com/your-username/trinity-ai-suite.git
git branch -M main
git push -u origin main
2

Enable GitHub Pages

Navigate to repository settings, choose the **Pages** menu, set the build source to **Deploy from a branch**, select `main` (or `/root`), and click **Save**. Your site goes live at `https://your-username.github.io/trinity-ai-suite/` in seconds.

3

Upload Installer Releases

Draft a new release on GitHub. Assign a tag (e.g. `v3.1`), upload your compiled `.exe` files from `05_Installers/` as assets, and hit publish.

4

Link Releases to Web Buttons

Update links in the download cards to use the direct release download endpoints to secure stable redirection:

<!-- Example direct link pointing to release assets -->
<a href="https://github.com/username/repo/releases/download/v3.1/Trinity_AI_Patch_v3.1.exe">

Frequently Asked Questions

Everything you need to know about setting up and running this free infrastructure.

Are there file size limits on GitHub Releases?
Individual release assets can be up to 2GB in size, which is significantly more than required for compiled PyInstaller executables (typically ~20MB to 50MB).
Will users get security warnings when downloading the EXEs?
Since the Python executables are compiled locally and not signed with a commercial certificate, Windows SmartScreen may show a warning. Users can click "More info" and then "Run anyway" to execute the utilities.
Is GitHub Pages secure and private?
Public repositories on GitHub Pages are public. If you need private downloads restricted to specific operators, you will need to utilize access tokens or basic credentials via alternative setups.
